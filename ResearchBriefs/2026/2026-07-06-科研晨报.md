# 科研晨报 2026-07-06

## 今日主线

今天继续按三条主线组织：**具身模型、场景理解模型、生成感知模型**。为避免和最近 7 天简报重复，本期避开已经覆盖过的 Embodied.cpp、VLAFlow、RegimeVGGT、FastPano3D、EmbodiedSplat、Guided Action Flow、The Moving Eye、SegVGGT、Bridge-WA、CoFL-S，以及更早的 DART、ASPIRE、Qwen-VLA、3DThinkVLA、ReactVLA、RoboSemanticBench、World Pilot 等条目。

今天的重点不是再堆一个新 VLA，而是看几个更底层的问题：**真实机器人评测应该怎么统计，长程任务中 VLM planner 与 VLA controller 如何编排，VGGT 如何变成轻量空间推理模型，WAM/VLA 是否正在融合为 WLA，以及世界-动作模型能否仅靠合成数据走向真机。**

## 5条简报

### 1. PhAIL：VLA评测不能只报固定超时成功率，要看 time-to-success 分布

**一句话结论**：PhAIL 提出真实机器人 VLA 的分布式评测方法，把 time-to-success CDF 作为基本评价对象，并用 Human-Relative Throughput 与宏平均 KS 检验区分策略差异；作者在 Franka FR3 上评估 OpenPI π0.5、GR00T、ACT、SmolVLA 等公开 VLA，指出最佳 VLA 仍约比同工位人类慢 7 倍。citeturn625003view0

**为什么值得关注**：这篇直接击中“具身模型评测”的痛点。现在很多 VLA 论文仍然用 N≤25 的 rollout 和固定 timeout 成功率，很难可靠区分两个接近策略。PhAIL 的核心价值在于把“是否成功”和“多快成功”合成到同一个分布里，并且把 hard failure 作为 T=∞ 的 ghost event，而不是简单丢掉或截断。citeturn625003view0

**是否开源**：论文明确说 PhAIL 是 open real-robot benchmark，包含 dataset、per-rollout artifacts 和 end-to-end reference implementation；论文正文还说明有约 990 episodes、396 human-teleoperation reference rollouts，以及可浏览的公开 run-explorer artifact。citeturn625003view0

**所需算力**：不是训练型论文，主要成本是评测平台与 rollout 采集。硬件是 Franka FR3、Robotiq 2F-85、外部 RGB 与腕部 RGB 相机；统计上每个 model/object cell 约 25–30 rollouts 就能用宏平均 KS 比固定阈值成功率更有效地区分部分模型对。citeturn625003view0

**输入/输出**：输入是同一真实机器人评测工位上的策略 rollout、视频、遥测和事件标注；输出是 time-to-success CDF、HRT、KS 显著性检验、P-P plot、RMST、UPH 等评测结果。

**作者核心 insight**：VLA 评测不是单一 scalar ranking，而是 survival/time-to-event 问题。成功率、吞吐、失败率、恢复尾部是同一个 CDF 的不同投影；只报一个阈值成功率，会丢掉策略速度与稳定性差异。

**思路来源**：来自真实机器人 benchmark、统计功效分析、Kaplan-Meier/time-to-event 分析，以及近年来对 robot learning empirical science 的批评。前序瓶颈是实机 rollout 成本高，导致评测样本小、无置信区间、无成对统计检验。

**团队启发**：我们做 VLA 提速、红外/偏振/全景模态任务时，不能只报 success rate。建议建立自己的 **time-to-success + failure mode + human-relative throughput** 表，尤其适合透明抓取、插销、装配这类“成功但慢/失败但可恢复”差异很大的任务。

### 2. Hi-VLA系统研究：长程具身任务的关键可能是 planner-controller 接口，而不是单个模型更大

**一句话结论**：这篇系统研究把 Hierarchical VLA 统一到 options-style control 框架下，比较高层 VLM planner、低层 VLA controller、切换机制、观察/记忆表示等设计，并在短程、长程和推理密集任务中总结出比 flat VLA 或朴素层级系统更强的设计原则。citeturn154516view0

**为什么值得关注**：它对应“具身模型”的系统架构问题。现在很多长程任务失败，不一定是低层 policy 不会抓，而是子目标拆解、子目标终止判断、记忆读写、planner 与 controller 的接口不稳定。对我们后续做 WAM/VLA、VLN、全景空间记忆很重要，因为高层空间理解和低层操作之间必须有清晰协议。

**是否开源**：arXiv 页面只给出项目页面与视频入口，未在页面中确认代码、模型权重或数据公开；当前按“论文与项目页公开，代码/模型/数据未确认”处理。citeturn154516view0

**所需算力**：论文更像系统实验和设计消融，不是单一大模型预训练。算力取决于所用高层 VLM 和低层 VLA controller；组内可低成本复现其核心思想：固定一个 OpenVLA/π0-style controller，只改变子任务生成、终止检测、memory representation 和 retry policy。

**输入/输出**：输入是视觉观测、语言指令、历史记忆或任务状态；高层输出语言子目标或 options；低层 VLA 输出连续或离散动作；系统输出长程操作任务完成结果。

**作者核心 insight**：Hi-VLA 不是简单“VLM 负责想，VLA 负责做”。真正影响性能的是 planner 什么时候调用 controller、controller 如何向 planner 暴露状态、失败如何触发重规划、memory 中保留什么信息。

**思路来源**：来自 options framework、SayCan/PaLM-E/RT-2 后的 language planner + skill/controller 路线，以及近年来 flat VLA 在长程任务中的失败。前序瓶颈是很多层级系统设计是经验拼接，缺少统一变量控制。

**团队启发**：如果我们做“VGGT 场景理解 + VLA 操作”，不要直接把 VGGT 点云塞进 VLA。更合理的是定义中间接口：对象位置、可达性、遮挡、已尝试动作、失败状态、下一子目标；这些正是 Hi-VLA planner-controller interface 需要的内容。

### 3. Distilling 3D Spatial Reasoning：VGGT可以作为轻量3D VLM的视觉几何底座

**一句话结论**：这篇将 7B 级 3D VLM 的空间推理能力蒸馏到 2.29B student，并用 VGGT 作为 vision encoder；作者报告 8.7× 推理延迟降低、模型大小约 3× 缩小，同时保留 teacher 54–72% 的性能，并在 ScanNet 与 3D-FRONT 的 proximity/contact 任务上达到 68–72% accuracy。citeturn520865view0

**为什么值得关注**：它正好落在“场景理解模型：通过 VGGT 系列提升空间理解”这条线。过去 VGGT 主要被理解为 feed-forward geometry backbone，输出 camera/depth/point map/track；这篇把 VGGT 接到轻量 VLM，并用 latent scratchpad 形式的 Hidden CoT 做 3D scene QA，说明 VGGT 可以进入 spatial reasoning，而不只是重建。

**是否开源**：arXiv 页面未显示直接代码、模型或数据链接；当前按“论文公开，代码/模型未确认”处理。citeturn520865view0

**所需算力**：训练涉及 7B teacher 到 2.29B student 的多任务蒸馏，比从头训练轻，但仍需要 ScanNet/3D-FRONT 场景数据、VGGT feature 抽取和 VLM 蒸馏。推理端是主要亮点：2.29B student + VGGT encoder，比 7B 3D VLM 更适合实验室单卡部署。

**输入/输出**：输入是 3D 场景视觉信息与问题；VGGT 作为几何视觉编码器；student 联合输出空间描述、深度估计、目标检测与 3D scene QA 答案。

**作者核心 insight**：3D reasoning 不一定要依赖显式长 CoT 文本。可学习 latent tokens 可以作为 Hidden CoT，在内部形成空间推理 scratchpad，从而避免显式 CoT 数据不足和推理时文本生成开销。

**思路来源**：来自 LLaVA-3D/3D VLM、VGGT feed-forward geometry、多任务知识蒸馏和 latent reasoning。前序瓶颈是大 3D VLM 难部署，而直接小模型又缺少几何归纳偏置。

**团队启发**：这给了一个清晰方向：**VGGT + lightweight spatial VLM**。组内可以让学生做“空间关系/可供性/导航记忆”的小模型，而不是一开始就训练大 VLM；重点评测 left/right/front/behind、contact/support、可达性、遮挡、目标重定位等机器人真正需要的空间问题。

### 4. WLA：WAM 与 VLA 正在融合成“世界-语言-动作”统一模型

**一句话结论**：World-Language-Action Model 提出 WLA 类模型，把语言指令、图像和机器人状态作为输入，同时预测文本子任务、subgoal image 与机器人动作；其 WLA-0 原型有 2B active parameters，在 RTX 5090 上 40 ms/inference，并报告 RoboTwin2.0 Clean 92.94% 与 RMBench 56.5% 成功率。citeturn154516view1

**为什么值得关注**：这篇不是单纯 VLA，也不是单纯 WAM，而是在模型定义层面把 world modeling、language reasoning、action synthesis 合并。它说明“生成感知模型”未来可能不是独立模块，而是作为隐式 world prediction 影响 action generation；推理时 world prediction 可以关闭，也可以打开做 test-time scaling。citeturn154516view1

**是否开源**：arXiv 页面未显示明确代码、模型或数据下载；当前按“论文公开，代码/模型/数据未确认”处理。citeturn304212view0

**所需算力**：WLA-0 是 2B active parameters，推理端作者报告 RTX 5090 上 40 ms/inference。训练端需要 world modeling objective、World Expert、Action Expert，以及大量 egocentric video/robot data；对组内而言不适合全量复现，但适合做小规模模块化验证。

**输入/输出**：输入是 textual instruction、images、robot states；输出包括 textual subtasks、subgoal images、robot actions。核心是 AR Transformer 预测 next state，包含语义级意图和细粒度物理动态。citeturn154516view1

**作者核心 insight**：world prediction 不一定要在推理时显式生成完整未来。通过 meta-queries，可以让 world prediction 在训练中塑造 action representation，推理时关闭以省时；必要时再打开做 test-time scaling。

**思路来源**：来自 WAM、VLA、video world model、subgoal image prediction 和 AR 语言模型路线。前序瓶颈是 WAM 能建模未来但语言/长程推理弱，VLA 有语义 grounding 但物理动态建模弱。

**团队启发**：这篇和“生成感知模型 + 场景理解模型”高度相关。我们可以不做 2B 全模型，而做一个小版本：VGGT/online 3D memory 预测下一步 geometric/semantic state，action head 读取这个 state token；推理时默认关闭未来生成，只保留压缩后的 action-relevant world token。

### 5. Synthetic Priors for WAM Sim-to-Real：世界-动作模型第一次被系统推向零真实示教真机迁移

**一句话结论**：这篇基于 Cosmos Policy，把视频 diffusion world-action model 用合成先验训练后零样本迁移到真实 Franka；作者用 AnyTask motion planning pipeline 生成每任务约 800 条合成示教，覆盖 object lifting、drawer opening、pick-and-place，真机零真实示教平均成功率为 35%。citeturn154516view2

**为什么值得关注**：这是“生成感知模型”非常值得跟踪的方向。35% 成功率绝对值不高，但如果结论成立，它说明 WAM 不只是视频预测玩具，而可以通过合成数据和 domain randomization 进入真实 manipulation。这对我们低成本构建具身数据、避免大量真机采集很有启发。

**是否开源**：arXiv 页面未显示代码或模型链接；论文备注说这是 CVPR 2026 Embodied AI Workshop 接收论文，且是官方 world-action model zero-shot sim-to-real transfer 工作的早期结果，后续将 release。当前按“论文公开，代码/模型未公开，需跟踪 release”处理。citeturn154516view2

**所需算力**：训练基于视频 diffusion/Cosmos Policy，算力应高于普通 imitation learning；但数据侧每任务约 800 条合成示教，没有真实示教。推理端部署在 Franka 上执行真实 manipulation；组内可先用小模型/小任务验证“合成先验 + 少量真实校准”是否比纯真实小数据更好。

**输入/输出**：输入是仿真生成的视觉观察、语言/任务条件和机器人状态；模型输出 visuomotor actions。训练数据来自 extensive domain randomization 的仿真环境与 AnyTask 运动规划示教。citeturn154516view2

**作者核心 insight**：WAM 的 sim-to-real 难点不是能否在仿真中生成动作，而是能否让视频 diffusion policy 学到对真实视觉与物理扰动有鲁棒性的 action-conditioned dynamics。大量合成数据可以提供“物理变化先验”，即使真实数据为零。

**思路来源**：来自 Cosmos Policy、video diffusion policy、domain randomization、motion-planning-generated demonstrations 和 sim-to-real robotics。前序瓶颈是 WAM 通常在仿真或离线视频中展示，很少证明能零样本落到真实机器人。

**团队启发**：这可以和我们的具身模态结合：先在仿真中生成 RGB/IR/depth/normal 多模态合成示教，再用少量真机数据校准。尤其适合暗光/红外任务，因为真实夜间数据采集成本高，但仿真可以系统改变光照、反射率、热源和遮挡。

## 组会讨论题

1. **我们自己的 VLA 评测是否要从 success rate 改成 time-to-success CDF？** 对插销、装配、透明抓取来说，“慢但成功”“失败但可恢复”“直接不可恢复”是三种不同能力，不应被一个固定 timeout 成功率掩盖。
2. **层级 VLA 的中间接口应该是什么？** 是语言子任务、3D object memory、可供性图、局部 waypoint，还是 action-relevant geometric tokens？这决定 VGGT/空间理解模块如何接入 VLA。
3. **VGGT 的价值是否应该从重建指标转向空间问答/可供性指标？** 如果目标是场景理解，depth/point map 只是底层输出，真正要评测的是 support/contact、遮挡、相对位置、可达性和目标重定位。
4. **WAM 是否应该显式生成未来图像？** WLA 的 meta-query 设计提示：未来预测可以只在训练中塑造动作表征，推理时关闭；这对低延迟机器人部署很重要。
5. **合成数据能否优先用于新模态任务？** 红外、偏振、全景、暗光这类真实采集成本高的模态，可能更适合先用合成先验训练，再少量真机校准。

## 可延展选题

### 选题1：PhAIL-style Multimodal VLA Benchmark

建立一个小型真实机器人评测协议，把 RGB、红外、偏振、全景、触觉模态放入同一 time-to-success CDF 框架。每个任务不只记录 success，还记录完成时间、失败类型、恢复次数、接触异常和 human-relative throughput。

### 选题2：VGGT Spatial Token Interface for Hi-VLA

让 VGGT 不直接服务低层 action，而是服务高层 planner：输出 object-level spatial token，包括对象位置、支撑关系、接触关系、遮挡状态、可达性、历史观测时间。低层 VLA 只接收经过 planner 筛选的子目标。

### 选题3：Hidden-CoT Spatial VLM for Robotics

基于 Distilling 3D Spatial Reasoning 的思路，训练一个 2B 级以下的 lightweight spatial VLM，用 latent scratchpad 做内部空间推理。任务聚焦机器人需要的问题：哪个物体更近，哪个物体被遮挡，夹爪从哪边接近，插销孔轴线在哪里。

### 选题4：WLA-lite with VGGT World Tokens

做一个小规模 WLA-lite：VGGT/online 3D reconstruction 输出 world tokens，语言模型输出 subgoal/action hint，action head 输出连续动作。训练时预测下一状态，推理时默认不生成图像，只使用 world token 改善 action。

### 选题5：Synthetic Infrared / Polarization Priors for WAM

借鉴 WAM sim-to-real 路线，在仿真中构造红外/偏振/暗光合成数据，训练模型学习 RGB 看不清时的边界、材质、反光和透明结构先验。评测重点不是图像生成质量，而是抓取成功率、动作时间分布和 failure reduction。
