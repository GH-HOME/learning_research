# 科研晨报 2026-06-30

## 今日主线

今天的主线是：VLA 研究正在从“更大的机器人基础模型”转向“部署时可诊断的系统能力”。过去几期已经覆盖了 VLA 提速、几何增强、物理门控、latent WAM、3DGS 操作数字孪生、异步多模态与因果世界模型；今天避开这些已覆盖条目，重点补齐五个相邻但不重复的问题：**鲁棒性如何训练、语义是否真的进入动作、长程任务如何随阶段自适应融合、非RGB传感器如何作为工具调用、3DGS世界模型如何预测多刚体动力学**。

## 5条简报

### 1. STRONG-VLA：鲁棒性训练不能和干净任务拟合同锅炖

**一句话结论**：STRONG-VLA 的核心判断是，视觉扰动和语言扰动下的鲁棒性学习，与干净示教数据上的任务拟合存在优化冲突；因此它把微调拆成两阶段：先用由易到难的多模态扰动课程学习鲁棒表征，再回到干净任务分布做 task-aligned refinement。论文建立了包含 28 类文本/视觉扰动的 VLA 鲁棒性 benchmark，并报告 OpenVLA、OpenVLA-OFT、pi0 都有跨架构收益，其中 OpenVLA 在 seen/unseen perturbations 下分别提升最高 12.60% 和 7.77%。citeturn744366view5

**为什么值得关注**：真实机器人里，鲁棒性问题经常不是单一视觉噪声，而是视觉遮挡、光照变化、摄像头裁切、语音/文本指令噪声、上下文干扰同时出现。STRONG-VLA 把这些扰动分为 12 类 textual 和 16 类 visual perturbations，并强调 robustness 是闭环任务成功率层面的属性，而不是单步预测精度。citeturn211811view0

**是否开源**：arXiv 页面没有显示明确 GitHub/代码/权重/数据下载入口；当前记为代码、模型、数据公开状态未知。citeturn744366view5

**所需算力**：不需要从头训练 VLA，主要是对已有 OpenVLA/OpenVLA-OFT/pi0 做两阶段微调。训练开销相对普通 SFT 增加，原因是 Stage I 需要生成并训练多种扰动样本，Stage II 再做干净数据回调；保守估计在同一 backbone 下约为普通微调的 1.5–2.5 倍。推理阶段不改变模型接口，理论上没有额外推理开销。

**输入/输出**：输入为视觉观测、自然语言指令及其扰动版本；输出仍是机器人动作。方法改变的是微调数据分布和训练日程，而不是 action space 或 VLA 推理结构。

**作者核心 insight**：干净数据要求模型保留细粒度语义和视觉敏感性，扰动数据要求模型对无关变化保持不变性；如果直接混合训练，梯度目标可能互相冲突。先扩展扰动支持集，再用干净任务重新对齐，是比“所有增强一起训练”更稳的路径。

**思路来源**：延续 VLA robustness、adversarial/data augmentation 和 curriculum training 路线。之前方法多把鲁棒性看成静态增强目标，容易牺牲干净任务性能；这篇把问题转为 stage-wise optimization。

**团队启发**：很适合我们的真机部署。可以把偏振/红外/暗光/反光/遮挡/指令歧义整理成组内 perturbation suite：先测 OpenVLA/StarVLA 是否在这些条件下失效，再做两阶段 LoRA 微调，而不是直接扩大真实轨迹数量。

### 2. RoboSemanticBench：VLA会抓东西，不代表它真的理解了题目

**一句话结论**：RoboSemanticBench 专门诊断 VLA 的语义接地：机器人看到多个候选答案方块，必须先解数学/常识/泛化知识问题，再抓取正确答案对应的实体。论文发现很多 VLA 在 grasp success 上不差，但控制 grasp success 后，语义选择接近随机甚至低于随机，说明 pretrained VLM 的语义能力没有稳定传递到 action prediction。citeturn747238view4

**为什么值得关注**：现在许多 VLA benchmark 可以被视觉捷径、任务模板记忆或 instruction-action shortcut 解决，导致模型看似“听懂了语言”，实则只会执行常见轨迹。RoboSemanticBench 把低层抓取和高层语义选择拆开，用 Task Success Rate、Grasp Success Rate 和 normalized Semantic Grounding 三个指标诊断“能抓”和“抓对”之间的差距。其 GitHub 仓库已经公开，包含环境、数据生成、policy evaluation 脚本，并基于 RoboTwin 2.0 工作流。citeturn107738view0

**是否开源**：已开源 GitHub 仓库，MIT license；仓库包含 assets、code_gen、data、description、envs、gsm8k、mmluqa2、policy、script 等目录。预采集数据方面，仓库说明已在 Hugging Face 提供部分 RSB-Math-4 和 RSB-Math-10 的 RoboTwin 格式数据，更多 suite 和 LeRobot 格式计划逐步释放。citeturn107738view0

**所需算力**：如果只是评测已有 policy，算力主要是 RoboTwin/SAPIEN 仿真运行和策略前向；若重新收集 RSB-General 或 HardMath 轨迹，仓库提供 4 GPU sharded collection 脚本，说明大规模生成更适合并行仿真。训练 VLA 则取决于选用 backbone，RSB 本身更像诊断 benchmark，而非必须训练新大模型。

**输入/输出**：输入为包含候选答案块的桌面视觉观测和一个多选数学/常识/知识问题；输出为抓取并放置某个答案块的机器人动作。benchmark 包括 4-choice 和 10-choice 版本，10-choice 将随机命中率降到 10%，更容易暴露语义接地问题。citeturn107738view0

**作者核心 insight**：VLA 的“语义理解”不能只看语言回答或任务成功率，必须在闭环动作中隔离低层控制能力和高层语义选择能力。若模型能稳定抓起方块，却选错语义目标，问题不在抓取，而在语言语义没有进入动作决策。

**思路来源**：来自 embodied QA、VLA semantic grounding、RoboTwin 仿真评测和 VLM reasoning benchmark。前序 VLA benchmark 往往把语义和运动混在一个成功率里，无法区分“动作不会做”与“语言没听懂”。

**团队启发**：这可以直接变成组内 VLA 评测模板：在透明抓取、偏振感知、红外抓取之外，加一组“语义选择一致性任务”。例如同一桌面有多个外观相似物体，指令基于材质、功能、物理属性或先验知识，观察 VLA 是否真的使用语义线索。

### 3. S^2-VLA：长程操作失败可能来自“融合权重不随任务阶段变化”

**一句话结论**：S^2-VLA 针对长程操作中的误差累积，提出 State-Space Guided Adaptive Attention：模型维护一个 belief state 追踪任务进展，并动态调节视觉空间感知、语言任务意图和历史动作序列三类信息的融合权重。论文 2026-06-26 提交，强调尽管模型规模为 compact 2B，但在 LIBERO 和 SimplerEnv 长程操作 benchmark 上超过更大的 7B 级模型。citeturn122983academia2

**为什么值得关注**：长程任务不是简单把短程策略滚很多步。早期阶段需要更多语言规划与目标定位，中期需要视觉-动作闭环，接触阶段需要历史动作和本体一致性；固定权重融合会使模型在不同阶段使用错误信息源。S^2-VLA 把“任务阶段”显式压进 belief state，用动态 gating 让信息融合随执行进展变化。citeturn122983academia2

**是否开源**：arXiv 页面和检索结果未显示明确代码、模型权重或数据公开入口；当前记为未知。citeturn744366view0

**所需算力**：完整训练 2B VLA 仍是多卡级别；但该思想可以轻量化复现：冻结视觉语言 backbone，仅在 action head 前增加 state-space belief tracker 和 adaptive fusion/gating module。推理阶段额外开销主要是 belief state 更新和 gating 计算，通常远低于 VLM 主干前向。

**输入/输出**：输入为视觉观测、语言指令、历史动作序列和内部 belief state；输出为机器人动作。关键不是新增传感器，而是在不同阶段自适应选择“看视觉、听语言、信动作历史”的比例。

**作者核心 insight**：长程操作的核心瓶颈不只是模型容量，而是信息源的动态选择。固定 fusion 对短任务尚可，但任务越长，阶段变化越明显，错误注意力就会变成系统性误差累积。

**思路来源**：来自 state-space model、belief tracking、long-horizon manipulation 和 adaptive attention。它与昨天的 DAM-VLA 不同：DAM-VLA 关注多模态传感器频率不同，S^2-VLA 关注同一任务内执行阶段不同。

**团队启发**：可以用于插销、装配、开抽屉等多阶段任务。我们可以先不训练大模型，而是给现有 VLA 加一个显式 phase/belief head，输出 search、approach、contact、insert、verify 等阶段，再用阶段控制不同模态和控制器权重。

### 4. MuseVLA：多模态传感器不必全时输入，可以像工具一样按需调用

**一句话结论**：MuseVLA 的关键设计是把非 RGB 传感器变成 on-demand tools：给定任务指令和视觉上下文，模型先生成 sensor token 和 target description，决定调用温度、声音、雷达等哪种传感器以及关注什么目标；随后把异质传感读数转换成 grounded sensor image，作为统一中间表示与 VLA 融合并输出动作。论文报告真实 dexterous hand manipulation 中平均成功率 80.6%，覆盖 temperature-guided pick-and-place、audio-driven object search 和 radar-assisted hidden object retrieval。citeturn122983academia3

**为什么值得关注**：多数 VLA 仍以 RGB 为主，但很多物理属性无法从 RGB 稳定推断，如温度、内部/遮挡物、声音响应、雷达反射等。MuseVLA 不把所有传感器硬拼接，而是先做“传感器选择”，再把测量转成可被 VLA 消化的 grounded sensor image，这对多模态机器人系统更工程化。citeturn439908academia1

**是否开源**：arXiv 页面和检索结果未显示代码、模型权重、数据生成 pipeline 或真实机器人数据公开状态；当前记为未知。citeturn744366view1

**所需算力**：完整训练需要多模态数据合成和真实机器人验证，成本高于 RGB-only VLA。由于作者引入数据合成 pipeline，将 RGB 视频数据增强为 grounded sensor images，复现可先从模拟传感器入手。组内可在 8×4090 上训练小规模 adapter/LoRA；推理时只有被调用的传感器产生额外成本，优于所有传感器全时编码。

**输入/输出**：输入为 RGB 视觉上下文、任务语言和可用传感器集合；中间输出为 sensor token、target description 和 grounded sensor image；最终输出机器人动作。其接口更像“VLA + sensor tool calling”，而不是传统 early-fusion 多模态网络。

**作者核心 insight**：多模态不是“传感器越多越好”，而是任务驱动地选择信息增益最高的传感器。把异质测量转换为统一视觉化中间表示，可以降低每种传感器都重训 VLA 主干的成本。

**思路来源**：来自 tool-augmented LLM/VLM、多模态感知、active perception 和机器人传感器融合。前序 RGB-only VLA 受限于不可见物理属性，传统多模态 VLA 又常需要昂贵同步数据和固定输入接口。

**团队启发**：这与我们偏振/红外/触觉路线高度相关。偏振不一定每帧都输入，而应在“透明膜、反光、弱纹理、暗光、隐藏边界”这类任务中被模型主动调用；可以把偏振 AoLP/DoLP、红外图、触觉图统一转成 grounded sensor image，再接 VLA。

### 5. MRO-GWM：3DGS世界模型可以从“预测像素”转向“预测物体刚体运动”

**一句话结论**：MRO-GWM 提出 Multi Rigid Object Gaussian World Model，用 object-centric Gaussians 表示任意形状和多物体场景，再用 spatio-temporal transformer 根据历史 object Gaussians 和未来动作预测刚体运动；它被用于典型家居物体多物体交互和机器人末端执行器推动，并在仿真中评估 model-predictive control 的非抓取操作。citeturn747238view2

**为什么值得关注**：过去几期我们讨论了 3DGS 数字孪生、GaussianDream 和 latent WAM，但 MRO-GWM 的角度更物理：世界模型不必预测整段未来视频，也不必把场景看成无结构 Gaussian cloud；如果场景主要由刚体物体组成，可以将每个物体规范到 canonical frame，然后预测 SE(3) 刚体运动。这样比像素视频生成更接近机器人规划需要的状态变量。citeturn551627academia0

**是否开源**：arXiv 页面未显示 GitHub、代码、模型或数据下载入口；当前记为未知。citeturn744366view2

**所需算力**：训练成本主要由多视角重建、object-centric Gaussian 表示学习和时空 transformer 动力学预测组成。相比大视频世界模型，MRO-GWM 更适合单机/小集群复现：可先用合成多物体数据和固定数量高斯训练动力学预测器。推理时若 object Gaussians 已构建，MPC rollout 成本主要是 transformer 前向和刚体变换更新。

**输入/输出**：输入为历史时刻的物体中心 Gaussian 表示、未来动作序列和多视角重建得到的部分观测；输出为未来物体刚体运动/未来 Gaussian 状态。对象在 canonical frame 中表示，使运动可被描述为 rigid body transformation。citeturn747238view2

**作者核心 insight**：机器人操作中的许多短程物理变化可以被“物体形状 + 刚体位姿变化”解释，而不需要预测每个像素或每个高斯自由形变。object-centric 结构既保留 3DGS 对任意形状的表达，又把动力学预测压缩到物体级运动。

**思路来源**：来自 object-centric representation、3D Gaussian Splatting、model-based control 和 non-prehensile manipulation。前序视频世界模型容易过度关注外观生成；动态 3DGS 又常缺少动作条件和可规划动力学接口。

**团队启发**：这可以服务我们“大区域轻量化 NVS/3DGS + 机器人操作”的落地方向：静态背景用稀疏/压缩 3DGS，操作对象用 object-centric Gaussian + 刚体位姿更新。这样比全场景动态 3DGS 更轻量，也更适合接 MPC 或 VLA 的候选动作评估。

## 组会讨论题

1. VLA 鲁棒性微调应优先做数据增强、两阶段课程训练，还是推理时 safety/physics wrapper？STRONG-VLA 和 PhysVLA 可以形成“训练时鲁棒”与“推理时约束”的对照。
2. RoboSemanticBench 暴露的 semantic grounding gap 是否会出现在我们的具身任务中：模型会抓透明杯，但是否真的理解“拿起能盛水的那个”？
3. 长程操作中，阶段 belief 应该从视觉/动作历史自监督学习，还是由规则/状态机提供弱标签？S^2-VLA 和 DAM-VLA 的 belief/buffer 是否可以合并？
4. 多模态传感应全时融合还是按需调用？对于偏振/红外/触觉，哪些任务值得让模型主动选择传感器？
5. 3DGS世界模型到底该预测未来视频、latent subgoal、Gaussian prefix，还是 object-level SE(3) motion？不同选择对应不同算力和可解释性。

## 可延展选题

1. **组内 VLA Robustness Suite**：围绕暗光、反光、透明、遮挡、相机裁切、语音转写错误和指令干扰，构建 20–30 类 perturbation，复现 STRONG-VLA 式两阶段 LoRA 微调。
2. **Semantic Grounding for Physical Attributes**：参考 RoboSemanticBench，设计“材质/功能/物理属性选择”任务，如拿起反光最大、温度最高、能透光、能导电或最可能滑落的物体。
3. **Phase-aware VLA Adapter**：为插销/装配/抓取任务增加 belief state 和阶段标签，动态融合视觉、语言、动作历史、力觉/触觉。
4. **Sensor-as-Tool VLA**：把偏振、红外、触觉、深度作为可调用工具；模型先生成 sensor token，再把传感器读数转成 grounded sensor image。
5. **Object-centric Gaussian MPC**：用 RGB-D/多视角重建 object Gaussians，训练动作条件刚体运动预测器，用于推、拨、滑、放置等非抓取操作的候选动作评估。
