# 科研晨报 2026-07-03

## 今日主线

今天的主线是：VLA 和具身智能的竞争点继续从“单任务成功率”向“长程真实任务、非视觉物理模态、低成本域迁移、3D/4D预测式规划和自我修复技能库”迁移。过去7天已经覆盖了 VLA 提速、鲁棒性、3D thinking、世界模型闭环评估、3DGS预测式表示、异步多模态和动作/动力学先验等方向；今天避开这些已覆盖标题，重点看 2026-07-01 前后新出现的 5 条互补路线：长程双臂家具装配、从人类触觉视频迁移到机器人触觉、单条示教完成VLA域适配、可解码为3D格式的结构化4D latent世界模型，以及 code-as-policy 体系下的自主技能发现。

## 5条简报

### 1. FurnitureVLA：长程双臂装配的关键不是“更大VLA”，而是进度信号和稳定子任务边界

**一句话结论**：FurnitureVLA 面向真实尺度双臂家具装配，任务最长包含 7 个子任务和 1550 个控制步；它把完整任务拆成语义接地子任务，并让 VLA 同时预测动作和连续 progress signal，从而自动触发子任务切换，仿真平均成功率从 monolithic finetuning baseline 的 48% 提升到 80%，真实 Kinova Gen3 上最难任务仅下降 16%。citeturn834447view0

**为什么值得关注**：它把 VLA 从常见桌面短程 pick-and-place 推向真实尺度、长程、高精度、双臂协同任务。论文特别强调，子任务边界不放在“刚装配完成的接触状态”，而放在 post-retreat 的无接触稳定状态，因为接触状态对误差极其敏感，会放大下一阶段的初始分布漂移。citeturn834447view0

**是否开源**：arXiv 页面只给出 project page 信息，未确认代码、模型权重或数据集公开；当前按“论文公开，代码/权重/数据未确认”处理。

**所需算力**：论文使用 π0.5 作为 VLA backbone，在 8 张 NVIDIA L40S 上微调 40,000 steps，global batch size 为 64；推理在单张 L40S 上运行。它还为每类家具生成 500 条仿真示教，并对每类家具评测 100 次 rollout。citeturn834447view0

**输入/输出**：输入为多相机 RGB 图像、本体状态和当前子任务语言指令；双臂动作以连续绝对末端执行器空间表示，左右臂各 7 维，拼接为 14 维动作，并额外输出 1 维 progress signal，因此训练时的增强动作向量为 15 维。citeturn834447view0

**作者核心 insight**：长程任务不是把短程策略滚动很多次即可，必须显式处理“什么时候进入下一阶段”。progress signal 比外部手写状态机更贴近策略内部执行进度，而 post-retreat boundary 则把阶段切换放在误差不易放大的稳定状态。

**思路来源**：来自 ACT/π0 的 action chunking、FurnitureBench/家具装配、长程模仿学习中的 subtask decomposition 和 progress prediction。前序瓶颈是 monolithic VLA 在长程接触任务中会累积误差，且手写阶段切换难以适应实际执行偏差。

**团队启发**：插销、装配、透明膜贴合、抽屉/门等任务都可以采用“阶段进度头 + 稳定边界”的思路。我们不一定先做大VLA，而可以在现有 policy 上增加 progress head，显式预测 search、align、contact、insert、retreat 等阶段。

### 2. H-Tac / TTP：触觉预训练开始从“机器人小数据”转向“人类大规模接触视频”

**一句话结论**：这篇工作提出 H-Tac 数据集和 Transferable Tactile Pre-Training（TTP）：H-Tac 包含 160 小时第一视角人类视频、300+ 任务和 135k episodes；TTP 通过统一触觉与动作空间，将人类接触经验迁移到机器人精细操作，并利用 tactile expert 预测未来触觉以建模接触动力学。citeturn107796view0

**为什么值得关注**：过去几期我们讨论过多模态 VLA、触觉/力觉异步输入和 sensor-as-tool；这篇的新增点是从“机器人触觉数据太少”转向“人类触觉行为能否作为预训练来源”。它明确指出触觉传感能提供视觉难以可靠推断的力反馈，而现有触觉数据集规模小、接触覆盖窄，VLA 触觉后训练也常忽略动力学。citeturn107796view0

**是否开源**：arXiv 页面未显示明确 GitHub、数据下载或模型权重入口；检索未确认 H-Tac 数据集或 TTP 代码公开，当前记为未知。

**所需算力**：完整预训练需要处理 160 小时视频、触觉/动作对齐和未来触觉预测，属于中到大规模多模态训练；组内更现实的路线是先冻结视觉/语言 backbone，用小规模人手接触视频或触觉传感器数据训练 tactile adapter 和 future tactile prediction head。推理阶段额外成本主要来自触觉编码器和预测头，通常低于再跑一个大VLM分支。

**输入/输出**：输入为第一视角人类视频、触觉信号、动作/接触轨迹，以及后续机器人任务中的视觉-触觉-状态观测；输出为机器人精细操作动作，同时 TTP 内部预测未来触觉以保留接触动力学先验。citeturn107796view0

**作者核心 insight**：触觉不是普通的辅助模态，而是接触动力学的直接观测。若只在机器人小数据上后训练触觉分支，模型很难学到丰富接触模式；人类大规模接触行为可以作为触觉先验来源，但必须通过统一触觉/动作空间桥接人机差异。

**思路来源**：来自 visuo-tactile learning、human-to-robot transfer、VLA 多模态后训练和 future prediction。前序瓶颈是机器人触觉硬件不统一、采集成本高、数据规模远小于视觉语言数据。

**团队启发**：对我们偏振/红外/触觉路线很有启发：多模态不一定只能从机器人采集，可以让学生先用人手/第一视角采集“接触—视觉变化—材料属性”数据，再把其中与机器人可迁移的接触先验蒸馏到 VLA 或 action head。

### 3. DART：VLA环境迁移可以用“权重向量算术”做单样本适配

**一句话结论**：DART（Domain ARiThmetic）把 VLA 域适配写成权重向量算术问题：在目标环境中只采集一条示教，通过提取并添加 domain-specific weight component，把已学任务迁移到相机视角变化或相似机器人变化等新域；论文称其在仿真和真实实验中优于已有 one-shot VLA adaptation 方法。citeturn107796view4

**为什么值得关注**：真实实验室里最常见的问题不是“完全新任务”，而是相机移动了、桌面换了、Franka 换成 UR5e 或 RealMan、夹爪安装略有变化。DART 的价值在于试图把这些 domain shift 从“重新采多条示教再训练”压缩成“一条示教 + 权重差分”。citeturn107796view4

**是否开源**：arXiv 摘要写有代码链接，但仓库当前只有 README，明确写着 “Code coming soon”，没有 release；因此应标注为“仓库已公开，实质代码尚未释放”。citeturn168032view0

**所需算力**：DART 不是从头训练 VLA，而是基于已有模型做少样本域适配；训练成本主要来自目标域单条示教的微调/权重差分估计和奇异分量子空间对齐，显著低于多任务重训。推理阶段通常不增加额外计算，只使用适配后的权重。

**输入/输出**：输入为源域 VLA 权重、目标域单条示教、相机/机器人/环境变化后的观测与动作；输出为加入域特定信息后的适配权重。方法内部通过奇异分量对齐过滤噪声权重成分。citeturn107796view4

**作者核心 insight**：不同任务能力和不同域信息在参数空间中可能部分可分离；如果能从一条示教中准确抽取“域差异向量”，就可以像语言模型中的 task vector/domain vector 一样做权重运算，避免每个任务重新收集多条轨迹。

**思路来源**：来自 model soups、task arithmetic、LoRA/adapter 差分、机器人 sim-to-real/real-to-real 迁移和 VLA 少样本适配。前序瓶颈是常规微调需要多条目标域示教，而且容易把任务能力和域特定信息纠缠在一起。

**团队启发**：非常适合组内多机械臂、多相机项目。可以做一个小实验：同一抓取/插销任务在 RealMan、Franka、仿真和不同相机位姿下各采 1 条示教，测试 DART 式权重向量能否复用源域策略。

### 4. Structured 4D Latent Predictive Model：世界模型需要预测3D结构演化，而不是只预测2D视频

**一句话结论**：Structured 4D Latent Predictive Model 以多视角图像和文本指令为输入，在结构化 latent space 中预测未来3D场景动态，并可解码为多种3D格式；它作为 planner 生成未来场景，再由 goal-conditioned inverse dynamics module 转成可执行动作。citeturn107796view5

**为什么值得关注**：它直接回应了视频世界模型的一个核心短板：2D视频预测缺少显式3D几何结构，长程 rollout 容易多视角不一致、空间关系漂移。项目页强调该方法集成多视角图像与文本指令来预测未来3D dynamics，并在 novel-view generalization 中显示出更强3D一致性。citeturn168032view1

**是否开源**：已公开 GitHub 仓库，包含 inference、training、data generation pipeline、policy evaluation 相关文件；仓库 README 提供 Hugging Face 权重下载方式，包括 generator 和 inverse_dynamics checkpoint，并给出 ManiSkill/StackCube 数据准备、训练和评测命令。citeturn221306view0

**所需算力**：完整训练需要 ManiSkill demonstration、多视角场景观测、TRELLIS-compatible feature extraction、latent scene encoding、generator training 和 inverse dynamics training；仓库示例中的 feature extraction batch size 为 16。对组内而言，可先用其 released weights 做推理和 LIBERO/ManiSkill 小任务评测，再考虑在单任务上微调 generator 或 inverse dynamics。citeturn221306view0

**输入/输出**：输入为多视角图像、文本指令和当前场景结构 latent；输出为未来3D场景 latent/可解码3D表示，并由 inverse dynamics 模块输出动作。真实实验部分还展示了从 action-free demonstrations 中重建3D点云、注册预测的夹爪几何以恢复末端位姿，再用运动规划执行子目标。citeturn168032view1

**作者核心 insight**：机器人规划所需的未来不是像素级“好看视频”，而是能保持多视角一致和物理空间结构的未来3D状态。若 world model 的 latent 本身可解码为3D格式，规划器就能基于几何目标而非像素差异做决策。

**思路来源**：来自 video predictive model、3D diffusion/TRELLIS、latent world model、goal-conditioned inverse dynamics 和 model-based planning。前序瓶颈是2D视频预测在遮挡、视角变化和几何一致性上不足。

**团队启发**：这条线与我们的 3DGS/NVS/3D reconstruction 积累最贴近。可以把它改造成“局部4D latent planner”：只预测操作相关物体和夹爪附近的未来3D结构，避免全场景视频生成的高算力成本。

### 5. ASPIRE：机器人技能发现正在从“重试同一轨迹”转向“调试程序并沉淀技能库”

**一句话结论**：ASPIRE 是一个 code-as-policy 的持续学习系统：机器人执行任务后记录多模态 trace，自动诊断失败、修复控制程序、重新验证，并把成功修复沉淀为可复用技能库；项目页报告其在 LIBERO-Pro、Robosuite 双臂 handover 和 BEHAVIOR-1K 长程家庭任务中显著超过既有基线。citeturn107796view8turn661943view0

**为什么值得关注**：它和典型 VLA 不同，不是把所有能力压进神经策略，而是让 LLM/agent 编写和修复机器人控制程序。项目页强调 ASPIRE 的三部分：closed-loop robot execution engine、continually expanding skill library 和 evolutionary search；其中执行引擎会记录每个感知、规划、抓取和控制调用的输入、输出与视觉证据，用于定位失败和合成修复。citeturn661943view0

**是否开源**：项目页显示 Paper，Code 标注为 coming soon；当前代码尚未释放。citeturn661943view0

**所需算力**：主要成本不在GPU训练，而在大量 LLM 调用、仿真/真实 rollout、失败诊断和 evolutionary search。项目页也把 compute-intensive search loop 列为限制，说明扩展到大规模任务需要更便宜的推理或更高效的搜索。citeturn661943view0

**输入/输出**：输入包括任务指令、机器人API、执行轨迹、多模态日志、失败状态和历史技能库；输出是可执行控制程序、修复后的程序片段以及可复用技能条目。它不是直接输出连续动作，而是输出可被底层 primitives 执行的代码策略。

**作者核心 insight**：许多机器人失败并不一定要靠重新训练神经网络解决，尤其当系统已有稳定 primitives 时，失败诊断、参数修复、子程序重组和经验沉淀可能更高效。技能库让修复经验跨任务、跨仿真/真实、跨本体复用。

**思路来源**：来自 Code-as-Policies、Voyager/agentic continual learning、robot execution trace、LLM debugging 和 evolutionary search。前序瓶颈是单次test-time retry往往只修当前轨迹，不会沉淀成长期可迁移技能。

**团队启发**：非常适合我们20人团队做工程系统：把每次真机失败都记录为结构化 trace，要求学生提交“失败原因—修复策略—是否可沉淀为技能”的日志。长远看，技能库可以服务 VLA 之外的传统规划、3DGS环境重建、偏振/红外感知和抓取控制。

## 组会讨论题

1. 长程装配任务中，progress head、阶段状态机和 VLA hidden state 哪个更可靠？FurnitureVLA 的 post-retreat boundary 是否可以迁移到插销/装配任务？
2. 触觉预训练应优先来自机器人触觉数据，还是人类第一视角触觉/接触数据？H-Tac 的 human-to-robot tactile transfer 对偏振/红外模态有没有类比？
3. DART 的“域向量”能否和 LoRA adapter 结合，形成相机/机械臂/夹爪/桌面环境的可组合适配库？
4. 4D latent world model 与 3DGS/MRO-GWM 的区别在哪里：前者预测结构化场景 latent，后者更偏对象级显式状态；哪一个更适合低算力真机闭环？
5. ASPIRE 这类 code-as-policy 自我修复系统，是否能和 VLA 共存：VLA 负责低层连续动作，agentic skill library 负责失败恢复和长程任务编排？

## 可延展选题

1. **Progress-aware VLA for contact-rich assembly**：为插销、装配、开抽屉任务增加 progress head，并把阶段切换点放在 retreat/relax 等稳定状态，而不是接触完成瞬间。
2. **Human-centric multimodal pretraining**：参考 H-Tac，用人手操作透明膜、包装膜、反光物、软物体采集视频+偏振/触觉/力觉数据，再迁移到机器人操作。
3. **One-shot domain vector library**：围绕相机位姿、机器人本体、夹爪类型和桌面环境建立 domain arithmetic adapter 库，减少每次换平台的重新采集成本。
4. **Local 4D latent planner**：借鉴 Structured 4D，只预测操作相关局部对象与夹爪的未来3D结构，再用 inverse dynamics 或 motion planning 生成动作。
5. **Robot failure trace + skill library**：为组内真机实验建立统一日志：观测、动作、失败帧、原因标签、修复代码、是否可复用；逐步形成自己的 ASPIRE-style 技能库。
